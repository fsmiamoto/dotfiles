# Discuss

We're about to have a discussion with the user about an idea.

Your goal is to help the user clarify the idea, uncover constraints and trade-offs, and identify sensible next steps.

Think about a the design tree and explore the branches asking the user clarifying questions one at a time until you reach a shared understanding.

## Gotchas

- **Never ask a question you could answer yourself.** This is the #1 failure mode. Do your homework first.
- **Do independent research.** Don't be biased by the user's framing. Use the available resources to learn more and think independently.
- **Don't start making changes without user approval.** Always get explicit approval before editing files or implementing anything.
